# 皮肤包接口 v1

皮肤包只替换画面，不改变动作状态机、帧顺序、触发条件、播放速度或交互逻辑。以后更新多套服装时，只需为所有动作提供同位置、同尺寸的新帧。

## 目录结构

把皮肤文件夹放在 EXE 同级：

```text
小曦薇.exe
skins/
  summer-blue/
    skin.xml
    frames.zip
```

`skin.xml`：

```xml
<?xml version="1.0" encoding="utf-8"?>
<skin apiVersion="1"
      id="summer-blue"
      name="夏日蓝裙"
      developer="Anbunengsi"
      exclusiveAction=""
      archive="frames.zip" />
```

约束：

- 文件夹名必须与 `id` 完全一致；`id` 只能包含英文字母、数字、点、下划线和连字符。
- `apiVersion` 当前固定为 `1`。
- `archive` 当前固定为 `frames.zip`。
- `name`、`developer` 与可选的 `exclusiveAction` 最长 64 个字符。
- `exclusiveAction` 可省略或留空；非空时表示该皮肤为 r15 提供一套专属动作，属性值就是菜单中显示的动作名。这个元数据只负责命名和启用既定槽位，不能改写代码、触发条件或其他行。
- 皮肤被选择后，本地只保存皮肤 ID；原皮肤文件被删除或损坏时自动回退内置白裙。

## 运行时换肤过渡

合规皮肤会经过“人物 alpha 切片伪 3D 旋身 → 新造型沿人物由头到脚变装 → 行 21 第 7 帧 pose → 行 0 待机”的统一过渡。程序只裁切和变换人物轮廓，不会把整个窗口或全画布当卡片翻转。若皮肤缺少最终 pose，程序会退回该皮肤的普通待机帧；按本接口提供完整 176 帧时不会触发此降级。

候选归档先在独立缓存中完成安全校验和关键帧预热，过渡结束时才原子替换当前缓存。切换失败只会丢弃候选内容，不会留下半套皮肤。过渡期间再次选择皮肤时，运行时采用“只保留最后一次选择”的有界队列，避免连续点击造成无上限内存占用。

## 帧归档

`frames.zip` 根目录直接包含：

```text
r00/c00.png
r00/c01.png
...
r23/c07.png
motion/r00/c00-r00-c01.mtn
...
```

不要再包一层 `frames/` 文件夹。每张图必须为带透明通道的 `528×808` PNG。`motion/` 中是官方构建脚本生成的紧凑双向运动网格，不是图片。归档必须完整覆盖当前动作契约，只能包含约定帧和合法运动网格，不能包含其他额外文件、重复条目、绝对路径、`..` 路径或反斜杠路径。

## 运行时补间

皮肤包保存上表中的 176 张关键帧，并可附带官方构建脚本生成的 `motion/*.mtn` 运动网格。程序会在相邻关键帧之间按需形变单一不透明姿态，使每个完整动画循环至少包含 24 个显示阶段；不再使用两张完整人物的透明交叉淡化。动作总时长沿用 v3.0.1 的节奏，阶段数变化不要求重建皮肤包。运动网格只记录位移，不包含第二套图像；旧皮肤未提供网格时会采用单姿态兼容过渡，仍不会产生双人重影。请勿增加 `c08.png` 等额外帧。

行 22 和 23 的持续状态由程序控制：换装皮肤必须保持相同的进入、循环／保持和退出语义，才能保证单击退出时动作连续。

看鼠标使用 r09/r10 的 16 个方向，只在光标进入桌宠附近的近距离范围时触发眼神和轻微头部跟随；光标离开范围后运行时回到 r00，仅保留普通呼吸眨眼。皮肤作者不应把 r09/r10 画成大幅扭身或持续转头。

## 安全校验

程序加载皮肤前会校验清单、路径、压缩包总体大小、每个条目大小、解压后总量、帧尺寸和重复条目。所有校验失败均只影响该外部皮肤，不会影响程序启动或内置白裙。

当前上限与官方打包脚本一致：单个 ZIP 条目不超过 32 MiB，`frames.zip` 文件本身不超过 128 MiB，全部条目解压后总量不超过 128 MiB。XML 属性可以使用合法的单引号或双引号；打包脚本默认输出双引号。

发行版中的两个示例皮肤位于 `skins/linan-princess` 与 `skins/huang-chengzi`，其 `exclusiveAction` 分别为“公主拂袖行礼”和“橙子放学元气招呼”，可直接作为完整 176 帧与 r15 专属动作的实现参考。

## 制作建议

- 先复制内置动作的透明帧作为位置参考，再逐行替换服装与发型。
- 同一皮肤中要固定头身比例、脚底基线、人物大小和脸部特征。
- 看鼠标的 r09/r10 必须以 r00 待机为尺度基准；官方脚本会要求 16 个方向的身高比例为 97%–103%，脚底与下半身中心偏差不超过 4 px。
- r12–r14 是内置动作槽；外部皮肤若不打算实现专属动作，将 `exclusiveAction` 留空即可。若启用专属动作，应逐帧制作 r15 的 8 张连贯关键姿势，不要用同一张静态图重复占位。
- r16–r17 虽暂时保留，也必须提供合法的 8 帧占位；不要删除行或改变归档路径。
- 先在真实桌宠显示尺寸下检查脸部清晰度、裙摆闪烁、脚底漂移和边缘光晕，再打包。
