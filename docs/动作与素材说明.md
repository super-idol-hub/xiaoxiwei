# 动作与素材说明

Windows 互动角色始终沿用用户确认的 vivo 活动白色无袖荷叶短裙、黑色细带高跟鞋和长卷发造型。动作只借鉴公开内容所呈现的气质，再压缩成适合窄幅角色窗口的原创循环，不逐帧描摹节目视频，也不嵌入原节目音乐、台词、字幕或品牌标识。

## 近距离看鼠标

看鼠标动作使用 r09/r10 的 16 个方向，但只在光标进入 Windows 互动角色附近的近距离范围时启用。运行时先让眼神朝向光标，再附加轻微头部转向，并使用平滑与迟滞避免快速抖动；光标离开范围后回正到 r00，仅保留普通呼吸和眨眼，不会隔着整个屏幕持续扭头。

## v3.0.6 皮肤扩展动作

- 内置 vivo 白裙使用 r12、r13、r14，依次为“卖萌”“哈哈哈大笑”“哭一下”；r15 为“白裙星光亮相”。四套动作可从右键菜单触发，也会参与该皮肤的随机待机。
- r15 是当前皮肤的专属动作槽。内置白裙由程序固定命名；外置皮肤只有在 `skin.xml` 的可选 `exclusiveAction` 非空时，才会用该文字命名并启用 r15。
- “临安公主｜大奉打更人”的 r15 为手动持续动作“公主荡秋千”：0–2 上秋千，循环严格按“c2 低点 A → c3 向前高点 → c4 低点 B → c5 向后高点 → c2 低点 A”沿画面纵深前后交替；已修复旧循环在闭合边界连续两次向前的问题，并为 c5→c2 新增单主体运动网格。单击后会等待 c2 低点，再转入 c6–c7 下秋千。人物始终正面居中、视线朝前，不左右横移或回头；通过远近尺度、身体俯仰、腿部和裙摆滞后表现前后摆动。动作全程保持原红珊瑚色皮肤，只将公开剧照中的双绳雕花长椅、坐姿和摆动气质转化为 Windows 互动角色动作；不嵌入宫殿背景、其他角色、字幕或标识。
- “黄橙子｜如此可爱的我们”的 r15 为“橙子放学元气招呼”，仍是普通有限动作。两套动作不会被其他皮肤调用。
- r16、r17 保留给后续兼容扩展；r18–r23 仍是所有皮肤共享的六套公共待机动作。

## 动画合成

- 内置皮肤共有 176 张 `528×808` 透明 4K 关键帧，动作行保留 4–8 个真正不同的关键姿势。
- 运行时对相邻关键姿势做缓入缓出补间，并用脚底／轮廓锚点减小整个人物的双影位移。
- 实际播放的普通循环、手机循环、侧躺进入和醒来片段均达到至少 24 个显示阶段，并沿用 v3.0.1 的动作总时长；看鼠标的方向行保持即时切换。
- 补间帧使用有上限的缓存按需生成，不写入皮肤包，也不会改变后续换装所需的关键帧行契约。

## 拖拽反应与扩展动作行

- v3.0.6 不在拖拽或自动漫游中播放走路、慢跑和快跑，避免关键姿势缺少完整左右腿交替时出现单腿循环感。
- 鼠标移动达到真实拖拽阈值后，Windows 互动角色使用跳跃行的前段表现突然受惊；松手后从生气行的皱眉帧开始，连续播放抬脚、跺脚和抱臂。
- 旧版 r12–r17 移动行不再由运行时触发：内置皮肤的 r12–r14 已用于三套表情动作，r15 用于当前皮肤专属动作，r16–r17 保留；动作契约仍固定为 24 行、每行最多 8 帧和总计 176 张关键帧。

## 换肤过渡

换肤时不会翻转整张窗口画布。运行时只裁切人物 alpha 轮廓，用多条水平切片的宽度和横向位移制造人物原地伪 3D 旋身；新皮肤在旋身过程中沿人物从头到脚逐段替换，最后进入 pose，再平滑回到普通待机。这样透明背景保持稳定，变化集中在人物本身。
